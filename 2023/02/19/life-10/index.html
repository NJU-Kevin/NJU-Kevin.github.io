数据库概论漫谈

Created2023-02-19|Updated2023-02-19|Life
|Word Count:775|Reading Time:2mins|Post Views:

今天白天大多时间都在复习数据库，准备晚上的考试。复习的过程中，发现数据库其实是一门很实用，很应用的学科。而且和C++不同，数据库理论与实验并重，数据库中有很多概念，从数据库的结构，到数据模式，模型，再到具体的一些对象，例如数据库，数据表，事务，规则，约束，游标，函数，索引，视图，触发器，存储过程等等，每一个对象都有着其独特的作用和语法结构。

理论部分比较重要的有：E-R图和关系模型的转化，关系模式的规范化（第一范式为属性不可拆，第二范式为非主属性必须完全依赖与关系中的所有主属性；第三范式为关系模式中不能有传递依赖）。

实验部分的代码与C++有类似之处，但仍有许多不一样的地方，比较重要的有：建库，建表，查询语句（内连接，左外连接，右外连接，交叉连接，除法），对数据库，数据表的修改和删除，建立索引，建立视图，自定义函数（标量函数，表值函数，多语句表值函数，以及SQL中常用函数的用法），建立触发器（Insert，Delete，Instead of，After），建立存储过程（利用游标读取）

大致来看，内容还是有些杂的，并且不同的对象有不同的语法，但对于不同的对象的理解有助于我们记住相应的语法，例如，索引是建立在某张表的某一列上的，于是建立索引的时候就需要

create nonclustered index on student(sid)

而视图是一张虚拟表，于是建立视图时的代码为

create view view_1 as select ... from ... where ...

创建函数的时候，如果函数需要输入参数，需要用变量的形式列出并附上数据类型，例如

create function my_function(@score int)
returns char(2)/table/@ftable table(...)
as
begin
...
end

游标也有其格式，声明游标变量时，

declare @cur cursor
set @cur = cursor
for select...
open @cur
fetch next from @cur (into ...)
while @@fetch_status = 0
begin 
    ...
    fetch next from @cur (into...)
end
close @cur
deallocate @cur

触发器是建立在表上的，在写触发器的定义的时候，注意有两张很重要的表：Deleted表和Inserted表。当执行检查操作不允许时，要执行rollback transaction操作，例如

create trigger tri on student
after insert
as
if ...
    ...
    rollback transaction
else
    ...
return

具体的操作需要结合题目要求进行完成，写函数的过程其实与C++编程类似，循环的边值问题等常见的“坑”同样也是需要多加注意的地方，由于是纸质书写，所以无法运行，这就需要具备“人工编译运行”的能力，这一点也是与C++类似的地方，但总体而言，代码的思维量不及C++多，但细碎的地方，需要注意的小细节会更多，这么看，数据库实际上还是一门很友好的程序课。
